# Hackaton_TP_Team65
Репозиторий для хакатона по ТП команды 65
*Автоматическая сортировка корпоративной почты*

Программа предназначена для автоматической сортировки входящих писем по категориям. Она читает файлы из заданной директории, анализирует их содержимое, классифицирует каждое письмо и раскладывает по соответствующим папкам. Результат работы фиксируется в логе и отчёте со статистикой.

Установка и запуск:

Для работы требуется Python 3.9 или новее. Запуск программы выполняется через bash-скрипт: bash run.sh. Скрипт проверяет наличие директории data/inbox, запускает Python-приложение, выводит итоговый статус обработки и сохраняет лог в logs/sorting.log.

Принцип работы:

Архитектура приложения построена на объектно-ориентированном подходе, что позволяет изолировать различные этапы обработки.

На первом этапе класс Parser читает все файлы из data/inbox, пропуская пустые файлы, бинарные данные и остальное, что не является письмом. Из каждого корректного письма извлекаются семь частей: заголовок, дата, тема, отправитель, получатель, текст тела и вложения (части могут быть пустыми). Поиск этих частей ведётся по ключевым словам на русском и английском языках. Текст приводится к единому регистру и очищается от транслита, чтобы улучшить анализ. Теперь пистма готовы к распределению.

На втором этапе происходит инициализация ключевых слов. При подключении новой папки с письмами выполняется анализ частотности слов: отсеиваются стоп-слова, встречающиеся в большинстве писем, и выделяются значимые термины из словаря в config.py. Результаты проходят ручную верификацию, после чего формируется набор ключевых слов для каждой категории. На основе анализа тестовых писем были определены категории: ?

Третий этап — классификация. Класс Classifier для каждого письма проверяет наличие ключевых слов из всех категорий. Если слова найдены в нескольких категориях, выбирается категория с наивысшим приоритетом. Письма без совпадений попадают в "Другое". Результатом работы становится словарь, где каждому файлу сопоставлена категория.

Четвёртый этап — управление файлами. Класс FileManager создаёт внутри data/processed подпапки для каждой категории, перемещает туда письма и формирует два отчёта: summary.json со статистикой по количеству писем в каждой категории и details.csv с детальным списком всех писем, их категориями и найденными ключевыми словами. Весь процесс логируется: в лог попадают ошибки чтения, нераспознанные форматы и успешные перемещения.

Обработка ошибок и граничных случаев:

Система спроектирована так, чтобы быть устойчивой к нештатным ситуациям. Если файл не является письмом (бинарный, пустой или повреждённый), он пропускается, а в лог записывается ошибка. Если письмо не содержит ни одного ключевого слова, оно относится к категории "Другое". Если письмо содержит ключевые слова из нескольких категорий, выбирается категория с наивысшим приоритетом. При отсутствии директории data/inbox скрипт выводит сообщение об ошибке и завершает работу. Благодаря этому приложение не падает на любых входных данных.
